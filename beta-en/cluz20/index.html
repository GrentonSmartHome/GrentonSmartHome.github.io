What's New

v5.13.04 (build 2428A)

  • Adding a new feature for the CLU object ZWaveRouting - sets the possibility of using routing when sending commands from the CLU to the Z-Wave module. By default, the ZWaveRouting feature is set to Off.
  • I/O MODULE 8/8 version 2.0.1 - improvement in the area of ​​device communication on the TF-Bus (problem with the disappearing module).

v5.13.03 (build 2422)

  • Support for the GRENTON LED RGBW Z-Wave module.
  • Support for the INFIBITY RADIATOR THERMOSTAT Z-Wave module.
  • Support for the GRENTON RS232 CONTROLLER.
  • New virtual object MultiFanACThermostat(object for fan coil units):
    • Support from 1 to 3 fan speeds,
    • Fan operation in continuous or automatic mode,
    • Thermostat operation only in cooling mode,
    • Control of the delay in turning on or off the control output in relation to the fan output,
    • Compressor protection against too frequent switching on.
Note!
To use the MultiFanACThermostat virtual object, you must have Object Manager v1.10.0 (or higher).
Info!
Adding a new object introduced a regression in the area of ​​the Thermostat object. Known bugs: OnOutOff event not generated during CLU restart, and it is not possible to control ZWAVE_DOUT and DIMM objects that are the output (Control feature) of the virtual object.
  • Adding new methods for the CLU object:
    • StartZWaveExclusion - initiates Z-Wave module exclusion mode,
    • StopZWaveExclusion - stops Z-Wave module exclusion mode.
  • Updated StartZWaveDiscovery method - the command name has been changed to StartZwaveInclusion and the Unlimited option has been added for the Time parameter - the addition process is active until the Z-Wave module is added.
  • Added time zones: Kolkata, Tehran, Riyad, Lagos, Johannesburg.
  • Added information about downloading NTP time to system logs.
  • Added new features and methods for the ZWAVE_CONFIG object of the GRENTON RELAY Z-Wave, INFIBITY RADIATOR THERMOSTAT Z-Wave:
    • Repeaters - number of devices intermediating in communication between CLUZ and a given module,
    • RepeatersList - list of devices (NodeID) intermediating in communication between CLUZ and the module,
    • UpdateNeighbours - triggers the action of updating and rebuilding the Z-Wave network (number of neighboring modules, method of communication with CLUZ) for a given module.
  • Added a new feature for the ZWAVE_CONFIG object of GRENTON RELAY X2 Z-Wave, GRENTON ROLLER SHUTTER Z-Wave, GRENTON LED RGBW Z-WAVE modules:
    • Standalone - setting a rigid connection between the states of inputs and outputs in the module - control without communication with the CLU (a detailed description of the operation of the Standalone mode can be found in the technical documentation of the module).
Note!
By default, the Standalone feature is set to 1 (On) - after the process of adding the module to CLU Z-Wave and performing CLU Discovery, the module remains in Standalone mode, i.e. in the same state as after taking it out of the box.
  • Added a new feature for the ZWAVE_DIN objects of the GRENTON RELAY X2 Z-Wave module:
    • Standalone - Sets the internal connection between the given input and the specified output: -1 - default setting, 0 - disabled, 1 - input coupled to the OUT1 output, 2 - input coupled to the OUT2 output.
  • Added queuing of Get/Set/SetDefault commands sent to Z-Wave battery modules - sending configuration parameters from the ZWAVE_CONFIG object takes place after the next wake-up of the battery module, not when the method is called.
  • Adding a new state "N/A" for the Value feature in the ZWAVE_CONFIG object - displayed when the value of a given register has not been retrieved.
  • Removal of the mechanism for banning Z-Wave devices. The Banned feature remains in ZWAVE_CONFIG objects to maintain backward compatibility.
  • Improved stability of Z-Wave devices.
  • Removal of a regression introduced in version 5.12 which resulted in sporadic disappearance of TF-Bus devices (voltage restart required to restore module operation).
Note!
To be able to update CLU to v5.13.02, you must have Object Manager v1.10.0 (or higher).
