What's New

v0.14.7 (build 2614)

1 April 2026

Fixes:

  • Fixed a Lua script execution error when the script ends with a comment without a trailing newline.
  • Fixed CLU discovery being blocked in Google Chrome due to CORS policy (Private Network Access).
  • timer object: Fixed inability to set feature values (e.g. timerDuration) via Runtime.
  • dimmer object: Fixed an issue where switchOn restored brightness to maximum instead of the last set value.
  • dimmer object: Fixed incorrect isOn being set to false when brightness is limited to its minimum.
  • led object: Fixed repeated calls to switchOnFor / switchOffFor not extending the active duration.
  • roller-shutter object: Fixed changePositionRetainSlats method not working.
  • roller-shutter object: Fixed reversePosition built-in feature not being set correctly.
  • roller-shutter object: Fixed incorrect behavior when calling changePositionPercent with parameter 0.
  • roller-shutter object: Fixed inconsistencies in event generation for shutter and slat movement.
  • roller-shutter object: Fixed missing return of currentValue, missing motionStatus change after blockMotion, and missing up/down feature changes during startCalibration.
  • roller-shutter object: Fixed incorrect slatsMovementTime values above 32.
  • dout / dout_lite object: Fixed missing isOn report after sending configuration; fixed incorrect powerValue and overload feature values.
  • digital-in object: Fixed missing digitalValue report after sending configuration with a feature value change.
  • analog-out object: Fixed missing onAnalogOutPercentChange event.
  • bus-voltage object: Fixed missing voltageValue reporting during rapid voltage drops and delayed first voltage value return after sending configuration.
  • analog-in object: Fixed incorrect onAnalogValueWarningLow/High event generation at CLU startup.
  • analog-in object: Fixed analogScaledValueOffset having no effect on analogScaledValue after sending configuration.
  • object-temperature-sensor object: Fixed incorrect temperatureCelsius value after sending configuration with a calibration change.
  • object-air-co2-sensor object: Fixed airCo2 and airVoc sensor values returning 0 when read from Lua scripts.
  • Fixed incorrect decimal values for numeric features across dimmer, led, roller-shutter, digital-in, analog-out, analog-in, and bus-voltage objects.

v0.13.3 (build 2613)

25 March 2026

New features:

  • Added the ability to set object feature values from Lua scripts.
  • Added support for log level selection in Lua diagnostic functions: log_trace, log_info, log_debug, log_error, log_fatal.

Fixes:

  • Fixed missing reporting of object events and states after adding a new module to the configuration.
  • Fixed incorrect dynamic outlet handling — events from objects not selected in the outlet were triggering the controller.
  • Fixed methods not working via MQTT after a voltage restart of the CLU.
  • Fixed missing reporting of module feature changes in runtime when calling the set$feature method.
  • Fixed missing MQTT publishing of controller feature states.
  • Fixed missing reaction to self-feature increment in controller scripts.
  • Fixed an issue where CLU treated all protocol-matching objects as selected in an outlet when no objects were explicitly chosen.

Improvements:

  • Optimized the script engine — improved performance of outlet selection resolution.
  • Optimized CLU startup time.

v0.10.2 (build 2609)

02 March 2026

Fixes:

  • Fixed incorrect controller behavior in Runtime.
  • Virtual object scheduler: Fixed an issue where onAnalogValueChange / onAnalogValueRise events were not generated after adding the object and sending the configuration.
  • Fixed missing MAC address and IP address return for CLU.
  • Fixed an issue allowing values outside the defined min/max range to be set.
  • Fixed an issue allowing invalid configuration values for features with predefined value sets.
  • Fixed an issue allowing configuration values of an incorrect data type to be set.
  • Object roller-shutter: Fixed an issue where calling the startMotion method with the motionTime parameter produced no response.

Improvements:

  • Changed diagnostic log timestamps from seconds to milliseconds.

v0.9.0 (build 2607)

16 February 2026

Fixes:

  • Fixed an issue that prevented establishing a connection to the CLU in a specific scenario (a GM refresh was required).
  • Fixed an issue where logs were not generated for objects inside a static outlet of an object (e.g., a humidity sensor within a multisensor).
  • Fixed an issue where a method in the script was not being invoked for objects inside a static outlet of an object (e.g., showError() for buttons[1]).

Improvements:

  • Optimized the CLU initial startup time.

v0.8.4 (build 2605)

09 February 2026

Fixes:

  • roller-shutter protocol: Fixed an issue where the shutter did not stop when the moveUp, moveDown, or startMotion methods were called again.
  • roller-shutter protocol: Fixed an issue where the startMotion method did not react when called with the motionTime parameter.
  • Fixed the behavior of the sourceObjectIndex block (outlet object index, e.g. the index of a pressed panel button).
  • Fixed an issue where dynamic outlet selection did not distinguish between the object-touch-panel-4b and object-touch-panel-8b protocols.
  • Fixed an issue where an outlet could select itself — the controller now ignores itself.
  • Improved outlet auto-selection logic — AND logic between categories and OR logic within a single category.

v0.8.0 (build 2605)

02 February 2026

New features:

  • Added support for logical operations on selections (adding, subtracting, multiplying, and dividing selections in Lua code).

Fixes:

  • Fixed an issue where the loc tag category was missing after a CLU factory reset.
  • Added support for the sourceObjectIndex block (outlet object index, e.g. the index of a pressed panel button).

v0.3.0 (build 2550)

15 December 2025

Fixes:

  • dimmer object: Fixed generation of the onIsOnChange event.
  • dimmer object: Fixed generation of the onDimmerBrightnessRise / onDimmerBrightnessLower events when calling methods other than switch/on/off.
  • dimmer object: Fixed setting the brightnessPercent value after calling the changeBrightnessPercent / changeDimmerBrightness methods.
  • led object: Fixed reporting of the distributedLogicGroup attribute.
  • led object [Led FM]: Fixed value changes when calling the switch/on/off methods.
  • led object [Led FM]: Fixed returning the brightnessPercent / ledBrightness values after calling the changeBrightnessPercent / changeLedBrightness methods.
  • led object [Led FM]: Fixed operation of the increaseBrightnessPercent and decreaseBrightnessPercent methods.
  • roller-shutter object: Fixed operation of the blockMotion and startCalibration methods.
  • dout object: Fixed returned value of the powerValue feature.
  • switch-for-duration protocol: Fixed switching on/off for a specified time when the provided parameter is of type int.

Known issues:

  • Connection issue with CLU if the Touch Panel 4B / Touch Panel 8B was added using New Project — a system restart is required for the connection to work properly.
