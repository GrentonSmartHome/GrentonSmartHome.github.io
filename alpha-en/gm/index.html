What's New

v0.6.8

30 March 2026

New features:

  • Redesigned action tiles in scripts for on-off devices, dimmers, LEDs, and roller shutters — replaced the "Set" and "Switch" tiles with a single "Change State" tile offering Turn On/Turn Off/Toggle options. Added a "Set brightness" tile (0–100% slider) for dimmers and LEDs, a "Set position" tile (0–100% slider), "Set slats position" (0–90° slider), and "Start motion" tile (Alternately/Up/Down) for roller shutters.
  • Added a new controller Presence Controller.
  • Added the ability to assign a virtual object to a specific CLU using the "Bind to CLU" option in the object properties.
  • Added the ability to add multiple modules and objects from the library in bulk — up to 20 items of the same type at once.
  • Added automatic label generation in the Overview view.
  • Added section headers (Events, Outlets, Scripts) to the object tree in the Editor tab.

Fixes:

  • Fixed false detection of unsaved changes when switching between Blockly and text-based scripts.
  • Fixed an incorrect timeout error when running long-running scripts — a success message is now shown after receiving a response from the CLU.
  • Fixed object name generation when creating an object linked to hardware — the name now uses the module's serial number (SN) instead of a UUID.
  • Fixed repeated appearance of the "Adding software modules" dialog when new modules were detected after a previous batch was added.

Improvements:

  • Improved notifications — extended display duration and added the ability to click a notification to open the related dialog.
  • Improved readability of the object properties list — adjusted font size and line spacing.

v0.6.7

24 March 2026

New features:

  • Added the ability to create an admin account during project initialization in the Wizard.
  • Added the ability to run a script directly from the editor tree.
  • Added a virtual object `Event Scheduler.
  • Added the ability to select the log level in debug blocks in scripts.
  • Added the ability to delete multiple modules at once in the Hardware tab.
  • Added highlighting of related elements in the Hardware tab when clicking on a association.
  • Buttons in the Smart Panel object are now displayed directly under their corresponding pages in the object tree.
  • Panel objects in the main Editor tree are now displayed directly under the panel object, without additional grouping.

Fixes:

  • Added validation for the module label — character limit and support for special characters.
  • Fixed the display order of features, methods, events, and outlets — now shown in protocol order instead of alphabetically.
  • Fixed validation of input fields in the Overview tab.
  • Fixed the boolean value selector in object property fields.
  • Fixed an issue where CLU returned an empty configuration response during Discovery — added automatic retry.
  • Fixed button sorting in the Smart Panel object.

Improvements:

  • Improved performance of the Diagnostics view.
