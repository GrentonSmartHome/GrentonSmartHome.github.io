Changelog

v5.12.01 (build 2230)

29 August 2023

  • Persistence of user features.
  • System configuration of electrical network parameters for modules: Relay x4 DIN, Relay x2 DIN, I/O 8/8 DIN, I/O 2/2 FM.
  • Roller Shutter x1 DIN and Roller Shutter x3 DIN version 3.2.4 - improvement in the area of device persistence.
  • Support for Z-Wave Aeotec Smart Switch 7.
  • Support for Z-Wave Fakro: ZWMR24, ZWS230, ARZ ZWR and AMZ ZWM.
  • Improvement of event generation of SunriseSunsetCalendar.
  • Improvement of cloud connection.
  • Improvement of diagnostic logs.
Note!
To be able to update CLU to v5.12.01, you must have Object Manager v1.8.0 (or higher).

v5.10.03 (build 2248)

5 December 2022

  • Support for diagnostic logging using telnet. Added new method (SetTelnetLogLevel) and feature (TelnetLogLevel) for setting log level.
  • Support for Roller Shutter x1 DIN in 3.2.1 version:
    • Maintaining the value of the Position and LamelPosition features after restart (persistence),
    • Additional protection against too short time intervals,
    • Stability improvements and bug fixes.
  • Support for Roller Shutter x3 DIN and Roller Shutter FM in 3.2.1 version:
    • Maintaining the value of the Position and LamelPosition features after restart (persistence),
    • Reversing of Roller Shutter direction,
    • Reversing of range of shutter position (0-100% for 100-0%),
    • Compensation of shutter actuation time,
    • Features that determine the upward and downward movement time of the roller shutter,
    • State of Roller Shutter calibration,
    • Option to specify target lamel position for SetPosition method - Default (lamel position in line with the roller movement), Previous (previous lamel position) or chosen by user,
    • Option for Calibration direction,
    • New events OnPositionChanged and OnLamelPositionChange,
    • Additional protection against too short time intervals,
    • Stability improvements and bug fixes.
  • Support for Z-Wave Aeotec Dual Nano Switch ZW132-C module.
  • Improvement of Thermostat virtual object:
    • Fixed generation of OnOutOn, OnOutOff oraz OnChange events,
    • Optimized ControlOutValue change time.
  • Improvement of Presence Sensor virtual object:
    • Changed of behavior after object locking,
    • TimeFromLastPresence counts if movement is detected or object is locked.
  • Improved State feature of CLU module,
  • Improved state of module after firmware upgrade.
  • Improvement of the TFBus module communication mechanism.
Note!
To be able to update CLU to v5.10.03, you must have Object Manager v1.6.1 (or higher)

v5.09.02 (build 2208)

17 August 2022

  • Support for the Roller Shutter x1 module in version 3.1.3 - improved settings of the lamel position.

    • Air humidity sensor,

v5.05.05 (build 2014)

24 April 2020

  • New virtual object Push – object enabling configuration of sending push notifications to mobile devices with the myGrenton application. It is possible to create up to 64 virtual objects. It has an implemented queue system that allows you to send a notification at a specified time interval (a maximum of 10 notifications can be queued per Push object).
    • New methods / features for Push object:
      • (Set)Message, (Set)Title – allows to set the content of the header and the message of the notification.
      • ClearMessage, ClearTitle – clears title and message buffer.
      • Send – sends push notification.
      • (Set)Interval – interval between push notifications.
      • LastTimeSend – return send time of last push notification.
    • New events for Push object:
      • OnSend – occurs when a push notification is sent.
      • OnOverflow – occurs when the outgoing buffer overflows.
Note!
To be able to take full advantage of push notifications, you must have Object Manager v1.3.3 (or higher) and the (Android) myGrenton v1.1.9 (or higher) or (iOS) myGrenton v1.3.0 (or higher) application. In addition, the use of cloud must be enabled on CLUZ (CloudConnection = true)
  • Unification for module serial number in config,json file.
  • Support for new Heimann Smokre Sensor module.
  • Roller Shutter module: State are set by slave device report.
  • Getting Hardware Version from TFBus modules.
  • Stability improvements and bug fixes.

v5.04.11 (build 1949A)

11 December 2019

  • CLU time setting using the NTP server including time zone and time change (winter/summer). The time is automatically synchronized with the NTP server. Two new CLUZ features are used for configuration:
    • NTPServer – allows you to enter an IP address (or domain name, eg.: pool.ntp.org) of an NTP server,
    • TimeZone – setting the CLUZ time zone - 22 zones are available.
Note!
Downloading time from the server requires that CLUZ must be connected to the Internet.Due to changes introduced in the area of time / date setting, there was also a change in the virtual measurement of media. The measurements taken are now recorded with the UTC time.*
  • The ability to change the voltage and frequency values used by the 2.0 series modules to measure the current/instantaneous power of the receiver. New features are used for configuration:
  • DefaultVoltageValue – default value of voltage defined in devices,
  • VoltageFrequency - voltage frequency in the power grid. > Note! | The settings only apply to devices in the 2.0 series!*
  • Support for new Z-Wave modules - enables cooperation with the following Z-Wave modules in the given range:
