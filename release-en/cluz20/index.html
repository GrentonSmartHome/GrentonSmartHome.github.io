Changelog

v5.07.02 (build 2120C)

16 June 2021

  • Support for New Virtual Objects - SunriseSunsetCalendar, PresenceSensor.
  • Add features responsible for monitoring the CLU supply voltage - BusVoltage, BusVoltageSensitivity, MaxBusVoltage, MinBusVoltage.
  • Add features for setting an DNS server address - PrimaryDNS / SecondaryDNS.
  • Support for Aeotec Nano Switch ZW139-C.
  • Support for OM diagnostic view.
  • Support for ROLLER SHUTTER x1 DIN and ROLLER SHUTTER FM in 2.1.2 version:
    • Percentage value of the shutter opening (0 - 100%).
    • Roller shutter lamel position (0° - 90°).
    • Support for operation of roller shutters in the range of 0-600 seconds.
    • Method for roller shutter calibration.
    • Option to Enable / disable the ability to control the roller shutter.
    • OnLamelOpen and OnLamelClosed event.
Note!
To be able to update CLU to v5.07.02, you must have Object Manager v1.4.1 (or higher)

  • Support for New TFBus module - DALI CONTROLLER DIN.
  • Support for New TFBus module - ROLLER SHUTTER x3 DIN.
  • Stability improvements and bug fixes.

v5.05.05 (build 2014)

24 April 2020

  • New virtual object Push – object enabling configuration of sending push notifications to mobile devices with the myGrenton application. It is possible to create up to 64 virtual objects. It has an implemented queue system that allows you to send a notification at a specified time interval (a maximum of 10 notifications can be queued per Push object).
    • New methods / features for Push object:
      • (Set)Message, (Set)Title – allows to set the content of the header and the message of the notification.
      • ClearMessage, ClearTitle – clears title and message buffer.
      • Send – sends push notification.
      • (Set)Interval – interval between push notifications.
      • LastTimeSend – return send time of last push notification.
    • New events for Push object:
      • OnSend – occurs when a push notification is sent.
      • OnOverflow – occurs when the outgoing buffer overflows.
Note!
To be able to take full advantage of push notifications, you must have Object Manager v1.3.3 (or higher) and the (Android) myGrenton v1.1.9 (or higher) or (iOS) myGrenton v1.3.0 (or higher) application. In addition, the use of cloud must be enabled on CLUZ (CloudConnection = true)
  • Unification for module serial number in config,json file.
  • Support for new Heimann Smokre Sensor module.
  • Roller Shutter module: State are set by slave device report.
  • Getting Hardware Version from TFBus modules.
  • Stability improvements and bug fixes.

v5.04.14 (build 2009)

26 February 2020

  • Cloud connection stability improvements.

v5.04.11 (build 1949A)

11 December 2019

  • CLU time setting using the NTP server including time zone and time change (winter/summer). The time is automatically synchronized with the NTP server. Two new CLUZ features are used for configuration:
    • NTPServer – allows you to enter an IP address (or domain name, eg.: pool.ntp.org) of an NTP server,
    • TimeZone – setting the CLUZ time zone - 22 zones are available.
Note!
Downloading time from the server requires that CLUZ must be connected to the Internet.Due to changes introduced in the area of time / date setting, there was also a change in the virtual measurement of media. The measurements taken are now recorded with the UTC time.*
  • The ability to change the voltage and frequency values used by the 2.0 series modules to measure the current/instantaneous power of the receiver. New features are used for configuration:
  • DefaultVoltageValue – default value of voltage defined in devices,
  • VoltageFrequency - voltage frequency in the power grid. > Note! | The settings only apply to devices in the 2.0 series!*
  • Support for new Z-Wave modules - enables cooperation with the following Z-Wave modules in the given range:
ModuleHandling range
Fibaro RGBW
(FGRGBWM-441 v2/5 EU)		- Reading / setting the status of individual output channels R, G, B, W (in the range of 0-255)
- Configuration parameters can be changed (Fibaro configuration interface)
Danfoss Living Connect
(EU HW: 00 FW:1.1)		- Setting the set temperature on the thermostatic head
- Enable / disable button lock
- Battery charge level reading
- Possibility to set / read module wake up time
  • myGrenton Application: o Increasing the buffer for connection to the cloud, o Reporting on value changes for:
    • User features,
    • features of the Calendar virtual object,
    • features of the Schedule virtual object,
    • features of the PID virtual object,
    • features of the Timer virtual object.
    • Analog In/Out - Unification of OnSwitchOn event generation for the AnalogIn and AnalogOut objects .
    • System stability improvements.

v5.03.06 (build 1927A)

08 July 2019

  • Support for remote CLU connection to the Grenton cloud via the myGrenton application interface.
  • New CLU features:
    • UseCloud - allows you to attach a connection to the cloud,
    • cloudConnection - specifies the CLU connection status from the cloud.
    • Support for new Grenton 2.0 modules - support for all currently sold Grenton 2.0 modules.
    • New CLU update mechanism - allows you to update both the Z-Wave CLU as well as the Grenton 2.0 modules.
    • Assigning numbers to Z-Wave objects with the same SN in different CLUs - adding the homeID of a given CLU to the name of the Z-Wave object.
    • Assigning an IP address when a given address is taken or 2 CLUs have the same addresses.
