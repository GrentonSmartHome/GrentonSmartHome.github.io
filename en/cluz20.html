v5.05.05 (build 2014)

24 April 2020

  • New virtual object Push ‚Äì object enabling configuration of sending push notifications to mobile devices with the myGrenton application. It is possible to create up to 64 virtual objects. It has an implemented queue system that allows you to send a notification at a specified time interval (a maximum of 10 notifications can be queued per Push object).
    • New methods / features for Push object:
      • (Set)Message, (Set)Title ‚Äì allows to set the content of the header and the message of the notification.
      • ClearMessage, ClearTitle ‚Äì clears title and message buffer.
      • Send ‚Äì sends push notification.
      • (Set)Interval ‚Äì interval between push notifications.
      • LastTimeSend ‚Äì return send time of last push notification.
    • New events for Push object:
      • OnSend ‚Äì occurs when a push notification is sent.
      • OnOverflow ‚Äì occurs when the outgoing buffer overflows.

Note!!

To be able to take full advantage of push notifications, you must have Object Manager v1.3.3 (or higher) and the (Android) myGrenton v1.1.9 (or higher) or (iOS) myGrenton v1.3.0 (or higher) application. In addition, the use of cloud must be enabled on CLUZ (CloudConnection = true)

v5.04.11 (build 1949A)

11 December 2019

  • CLU time setting using the NTP server including time zone and time change (winter/summer). The time is automatically synchronized with the NTP server. Two new CLUZ features are used for configuration:
    • NTPServer ‚Äì allows you to enter an IP address (or domain name, eg.: pool.ntp.org) of an NTP server,
    • TimeZone ‚Äì setting the CLUZ time zone - 22 zones are available.

Note! Time synchronization requires the CLU to have an Internet connection. Due to changes introduced in the area of time / date setting, there was also a change in the virtual measurement of media. The measurements taken are now recorded with the UTC time.

  • The ability to change the voltage and frequency values used by the 2.0 series modules to measure the current/instantaneous power of the receiver. New features are used for configuration:
  • DefaultVoltageValue ‚Äì default value of voltage defined in devices,
  • VoltageFrequency - voltage frequency in the power grid.

Note! The settings only apply to devices in the 2.0 series!

v5.03.06 (build 1927A)

08 July 2019

  • Support for remote CLU connection to the Grenton cloud via the myGrenton application interface.
  • New CLU features:
    • UseCloud - allows you to attach a connection to the cloud,
    • cloudConnection - specifies the CLU connection status from the cloud.
  • Support for new Grenton 2.0 modules - support for all currently sold Grenton 2.0 modules.
  • New CLU update mechanism - allows you to update both the Z-Wave CLU as well as the Grenton 2.0 modules.
  • Assigning numbers to Z-Wave objects with the same SN in different CLUs - adding the homeID of a given CLU to the name of the Z-Wave object.
  • Assigning an IP address when a given address is taken or 2 CLUs have the same addresses.
